# 业务内核统一(P11)+ iOS 平台隔离 · Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** App(RN)切换消费 `@pocket-code/client-core` 并删除三个冻结副本;iOS 隔离 `local` 模式(默认 relay、local 置灰、终端 Tab 不渲染),两平台共享同一业务内核。

**Architecture:** A 先 B 后:先做纯重构(client-core 加 `onTokenPersist` 透传 → App 切 import + 接线 → 删副本),每步 `test:all` 绿;再叠 iOS 行为分叉(平台默认值、Tab 过滤、Settings 置灰),分叉逻辑提纯为纯函数单测。Spec:`docs/superpowers/specs/2026-07-11-内核统一-iOS平台隔离-design.md`。

**Tech Stack:** TypeScript、React Native (Expo 54)、vitest(**不是 jest**)、pnpm workspace。

## Global Constraints

- 分支:`feature/p11-ios-isolation`(从 master 切出;执行者若发现已在此分支则直接工作)。
- **Android 行为零变化**:默认 `workspaceMode` 仍 `"local"`、终端 Tab 仍在且常驻挂载、relay 配对/token 持久化行为等价。
- **iOS 仅隔离 `local` 一种模式**;`server`(直连)与 `relay`(中继)完整支持;iOS 默认 `workspaceMode` 为 `"relay"`。
- client-core 公共 API 仅新增可选字段:`ConnectionConfig.onTokenPersist?: (token: string, machineId: string) => void`,其余不变(Web 消费方不受影响)。
- **明确不改**(spec §5 B4,防过度守卫):`localExecutor.ts`、`runtimeManager.ts`、`processManager.ts`、`deviceBackend.ts`、`localFileSystem.ts`、`FilesTab`、`PreviewTab`、`useAgent` 的 mode 分支逻辑、`pocket-terminal-module` iOS Swift stub。

- client-core 的 `main`/`types` 指向 `dist/`:**改 client-core 源码后必须 `pnpm --filter @pocket-code/client-core build`**,否则 app 侧 tsc 读到旧 dist。
- 验证门(每任务末尾适用其一或多):`pnpm --filter @pocket-code/client-core test`、`pnpm --filter @pocket-code/app test`、`cd packages/app && npx tsc --noEmit`(今日基线 0 错误,必须保持 0)、`pnpm test:all`。
- iOS 置灰文案(逐字):`本地终端仅 Android 支持`。

**Files:**

- Test: `packages/client-core/src/serverConnection.test.ts`

**Interfaces:**

- Produces: `ConnectionConfig.onTokenPersist?: (token: string, machineId: string) => void` —— T2 的 useAgent 靠这个字段注入 RN 持久化。

**背景(给零上下文实现者):** `ServerConnection` 在 relay 模式下内部构造 `RelayClient`,但现在没把宿主的持久化回调递进去 —— 配对 token 持久化会静默丢失。本任务把 config 的回调透传给 RelayClient。测试文件已有 `FakeWebSocket`(`vi.stubGlobal("WebSocket", FakeWebSocket)` 于 beforeEach,约 L69)、`makeConfig(overrides)`、`makeHandlers(overrides)` 三个现成辅助,直接复用。

- [ ] **Step 1: 写失败测试**

在 `packages/client-core/src/serverConnection.test.ts` 末尾(最后一个 `describe` 内部,与既有 it 平级;若文件是多个顶层 describe,放入与 relay 相关的那个,否则加在文件末尾新 describe)追加:

```ts
describe("onTokenPersist 透传", () => {
  it("relay 模式下把 config.onTokenPersist 透传给 RelayClient(updateToken 触发宿主回调)", () => {
    const persist = vi.fn();
    const conn = new ServerConnection(
      makeConfig({
        isRelayMode: () => true,
        getRelayOptions: () => ({ machineId: "m_1", deviceId: "d_1", token: "tok_0" }),
        onTokenPersist: persist,
      }),
      makeHandlers()
    );
    conn.connect();
    // connect() 在 relay 模式下创建 RelayClient(私有 ws 字段)。
    // 经其公共 API updateToken 断言回调被透传(行为断言,不翻私有 opts)。
    const relay = (conn as unknown as { ws: { updateToken(t: string, m: string): void } }).ws;
    relay.updateToken("tok_1", "m_1");
    expect(persist).toHaveBeenCalledWith("tok_1", "m_1");
  });
});
```

注意:beforeEach 已 stub 全局 WebSocket 为 FakeWebSocket,RelayClient 构造不会真联网。

// Tab 栏平台过滤(纯函数,不 import react-native —— 便于 vitest 直测)。
// iOS 沙箱禁 fork/exec,本地终端为 Android 专属 → iOS 不渲染 terminal Tab。
export const ALL_TABS = ["chat", "terminal", "files", "preview"] as const;
export type TabKey = (typeof ALL_TABS)[number];

export function tabsForPlatform(os: string): readonly TabKey[] {
  return os === "ios" ? ALL_TABS.filter((t) => t !== "terminal") : ALL_TABS;
}
